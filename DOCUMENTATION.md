## 1. Project Purpose

**Lughati** ("لغتي" — My Language) is a gamified Arabic reading and writing platform for young children. It combines:

- AI-generated personalised Arabic stories (illustrated, 3 pages each)
- Letter-by-letter lesson reading with writing canvas and AI correction
- Placement test to determine a child's Arabic level (1 / 2 / 3)
- AI-generated quizzes after each lesson (MCQ, Drag-Drop, Ordering) — generated by a single ExamAgent
- PDF import → structured lessons pipeline
- RAG (Retrieval-Augmented Generation) knowledge base for teachers
- Multi-role dashboards (Student / Parent / Teacher / School Admin / System Admin)

## 3. User Roles

| Role | Value | Description |
|------|-------|-------------|
| `Student` | 0 | Child — reads stories, writes, takes quizzes |
| `Parent` | 1 | Monitors child progress — does not create student accounts |
| `Teacher` | 2 | Creates students, lessons, views class analytics |
| `SchoolAdmin` | 3 | Manages school profile, assigns teachers, views school-wide reports |
| `SystemAdmin` | 4 | Full access — manages content, RAG, AI settings, users, schools |

Students authenticate with an **image PIN** (two photos selected at account creation), not a password. All other roles use email + password with JWT tokens.

### 4.4 Application Layer

#### Agents (`Application/Agent/`)

**`StoryAgent.cs`** — Main story generation pipeline:
1. Calls `QwenStoryGeneratorService` → generates Arabic story JSON (3 pages)
2. Passes to `QwenJudgeService` → validates content is child-appropriate
3. Calls `ComfyUiImageService` per page → generates illustrations
4. Saves story + pages to database

**`ExamAgent.cs`** — Unified exam generation + scoring (Quiz and Exam are the same agent):
1. Reads lesson sentences from database (quiz after story has been removed)
2. Calls `QwenExamGeneratorService` → generates questions: **MCQ**, **Drag-Drop**, **Ordering**
3. Caches result in `GeneratedExam` table keyed on `LessonId` — will not regenerate if a record already exists
4. Exam is shown immediately after the student finishes a lesson
5. Handles answer submission and score calculation with stars/progress saved to `StudentProgress`

**`WritingCorrectionAgent.cs`** — Writing evaluation:
1. Receives canvas image (PNG blob) + expected sentence
2. Calls `TesseractOcrService` → extracts Arabic text from image
3. Calls `ArabicSimilarityService` → computes similarity score
4. Returns feedback: isAccepted (≥60%), score %, extracted text, message

**`LessonGenerationAgent.cs`** — RAG-based lesson generation:
1. Queries `RagQueryService` with a topic
2. Builds lesson structure (letter, level, pages)
3. Calls image generation for lesson illustrations
4. Saves lesson to database

#### Interfaces (`Application/Interfaces/`)

**`IServices.cs`** — all service contracts:
- `IStoryGeneratorService` — generate Arabic story text
- `IExamGeneratorService` — generate exam questions
- `IJudgeService` — validate story content safety
- `IImageGenerationService` — generate images via ComfyUI
- `IOcrService` — extract text from images
- `ITextSimilarityService` — compare two Arabic strings
- `IPdfPageRenderer` — render PDF pages to images
- `IAiTextCleanupService` — clean OCR-extracted text
- `IPdfImportService` — import PDF as lesson
- `IAuthService` — register, login, JWT, create school admin

### 5.3 Features (Components)

#### Home (`/`)
Landing page with hero section, feature highlights, and pricing plans (مجاني / عائلي / مدرسي). Bottom section added in recent sprint.

#### Auth
- **Login** — email + password form, redirects by role (student/parent/teacher/school/admin)
- **Register** — adult user registration
- **Create Student** — parent or teacher creates a child account with name, age, level, image PIN selection. Supports `?returnTo=parent|teacher|school` query param for correct back-navigation.

#### Placement Test (`/test/*`)
Three-component flow:
1. **Welcome** — explains the test, Start button
2. **Questions** — 15 questions across 3 parts:
   - **Part 1 — Letter recognition:** student hears a letter sound and picks the matching Arabic letter. Questions of type "ما هو الحرف الذي تسمع؟" hide the letter graphic entirely — only a pulsing speaker icon is shown. Detection is automatic: if `questionText` contains "تسمع" / "صوت" / "تسمعه" the letter is hidden (`isAudioOnly` computed signal).
   - **Part 2 — First letter of a word:** image shows a word/object, student picks the first letter.
   - **Part 3 — Sentence comprehension:** ordering, meaning, and context questions.
   - Reads from API, falls back to 15 built-in mock questions if API is unavailable.
3. **Result** — shows assigned level (1/2/3) and per-part breakdown. Redirects to `/dashboard`.

#### Lesson Reader (`/lessons/:id`)
The most complex component. Key features:
- **Canvas drawing** — pointer events for touch + mouse. Canvas set up after lesson loads (not `ngAfterViewInit`) to avoid timing bug with `@if` block.
- **Letter watermark** — lesson's letter drawn at 78% canvas height, 13% pink opacity fill + dashed pink stroke, for child tracing.
- **Writing check** — submits canvas `toBlob()` → API `/api/writing/evaluate` → shows score, extracted text, expected sentence, accept/reject feedback.
- **Page navigation** — left/right arrows + keyboard `ArrowLeft`/`ArrowRight`.
- **Audio** — Web Speech API reads page sentence aloud.
- **Progress** — saves to `AppStateService` on every page turn.
- **On finish** — navigates to `/exam` for lesson quiz.

**Key bug fixed:** `page.pageId` not `page.id` (LessonPage interface uses `pageId`).

#### Exam (`/exam`)
Displays exam generated from last lesson. Uses `AppStateService.currentLesson()` to load the lesson's exam. Supports all 4 question types:
- **MCQ** — 4 option buttons
- **Matching** — left column picks, right column targets, auto-matches
- **DragDrop** — blank sentence with word chips to fill
- **Ordering** — word chips to place in correct order

#### Story Reader (`/books/:id/read`)
Page-by-page story reading with full-page illustrations, Arabic sentence display, audio playback. On finish → navigates to `/books/:id/quiz`.

#### Teacher Area
- **TeacherDashboard** (`/teacher/students`) — lists students, class stats. Teacher is the one who creates student accounts: `+ أضف طالباً` button navigates to `/auth/create-student?returnTo=teacher`
- **LessonCreate** (`/teacher/lessons/create`) — upload PDF or enter topic, trigger lesson generation
- **AiGenerator** (`/teacher/ai-generator`) — story generation tool for teachers
- **TeacherReports** (`/teacher/reports`) — student performance reports

#### Admin Area (System Admin only)
- **AdminRag** — knowledge document list, upload, delete, search RAG
- **AdminImport** — import PDF as lesson (letter, level, PDF file)
- **AdminPdfLibrary** — manage PDF library, generate embeddings
- **AdminContent** — view/manage stories and lessons
- **AdminAiSettings** — configure Ollama model, ComfyUI settings
- **AdminSubscriptions** — subscription statistics
- **AdminUsers** — user list, block/unblock
- **AdminSchools** — create school admin accounts (form: school name, admin email, admin password) → returns school code for teachers to use on registration

#### Parent Dashboard
Child summary cards, progress charts, streak, weak letters. Read-only monitoring view — the parent dashboard does not have a "create student" button. Student accounts are created by teachers.

#### School Dashboard
School-wide stats, teacher list, class performance overview.

## 6. AI Pipeline

### Story Generation Pipeline

```
1. User submits: character + theme + level
2. StoryAgent calls QwenStoryGeneratorService
   → Ollama POST /api/generate (qwen2.5:1.5b)
   → Prompt: system + user template from Application/Prompts/
   → Output: JSON { title, pages: [{sentence, imagePrompt}] }
3. StoryAgent calls QwenJudgeService
   → Validates output is child-appropriate
   → If rejected → regenerate (max 2 attempts)
4. StoryAgent calls ComfyUiImageService per page
   → POST workflow to ComfyUI
   → Poll for completion
   → Save image to wwwroot/images/
5. Save Story + StoryPages to SQL Server
6. Return story ID to client
```

### Exam Generation Pipeline (Unified — Quiz and Exam are the same thing)

```
1. Student finishes a lesson → client calls POST /api/exam/generate/lesson/{lessonId}
2. ExamAgent checks: does GeneratedExam exist for this lessonId?
   → YES: return cached exam immediately (no LLM call)
   → NO: continue to step 3
3. ExamAgent reads lesson sentences from DB
4. Calls QwenExamGeneratorService
   → Prompt includes sentences + instruction to produce questions
   → Output: JSON array with three question types:
       • MCQ       — pick the correct answer from 4 options
       • Drag-Drop — fill the blank in a sentence using word chips
       • Ordering  — arrange shuffled words into a correct sentence
5. Saves GeneratedExam (keyed on LessonId) + Questions to DB
6. Returns Exam with all questions to client
7. After student submits answers → save score + stars to StudentProgress
```

> Note: The quiz after story reading has been removed. Exams only appear after lessons.

### Writing Correction Pipeline

```
1. Student draws on canvas → toBlob() → multipart POST /api/writing/evaluate
   Fields: lessonId (Guid), pageId (Guid), childName (string), image (PNG file)
2. WritingCorrectionAgent:
   a. TesseractOcrService extracts Arabic text from image
   b. QwenOcrCleanupService cleans up extracted text
   c. ArabicSimilarityService computes score vs expected sentence
   d. isAccepted = score >= 60%
3. Saves WritingAttempt to DB
4. Returns: { isAccepted, similarityScore, message (Arabic), extractedText, expectedSentence }
```

### RAG Pipeline

```
INGESTION:
1. Admin uploads document (PDF/PPTX/image) to /api/rag/ingest
2. RagIngestionService:
   a. DocumentProcessor extracts text + images
   b. ArabicTextChunker splits into chunks (400 chars, 60 overlap)
   c. OllamaEmbeddingService embeds each chunk (nomic-embed-text)
   d. ChromaVectorStoreService stores in collection "arabic-edu"
   e. Saves KnowledgeDocument record to SQL Server

QUERY:
1. Client sends query text to /api/rag/search
2. RagQueryService:
   a. Embeds query with OllamaEmbeddingService
   b. Queries ChromaDB for top-K similar chunks
   c. Returns chunks with similarity scores
```
